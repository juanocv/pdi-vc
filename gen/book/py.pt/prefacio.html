Prefácio

Este livro é um projeto em construção sobre Processamento Digital de Imagens (PDI) e Visão Computacional (VC), concebido como material didático interativo para cursos de graduação e pós-graduação em Computação, Engenharias e áreas afins.

📘 Destaque: Este livro segue a metodologia de Zampirolli et al. (2025) — uma versão estendida de Zampirolli et al. (2024), que recebeu o prêmio de melhor artigo na trilha de “Recursos e Ambientes Educacionais” na EduComp 2024 — com a biblioteca morph.py, desenvolvida pelo autor.

Um livro vivo

Este não é um livro estático. O conteúdo está em constante evolução: exemplos são refinados, novas seções são adicionadas e abordagens pedagógicas são aprimoradas com base no feedback de alunos e professores. Por isso, tratamos este material como um projeto inicial — uma base sólida que continuará crescendo.

Uso de ferramentas de Inteligência Artificial

Este livro foi produzido com o apoio de ferramentas de Inteligência Artificial (IA) para aprimorar a qualidade do texto e do código. A versão inicial dos capítulos passou por um processo de revisão e correção utilizando múltiplos assistentes de IA, incluindo DeepSeek, Gemini e Claude. Por meio de prompts iterativos, essas ferramentas ajudaram a:

  • Refinar a clareza e a precisão das explicações;
  • Detectar e corrigir erros de sintaxe, lógica e digitação nos exemplos de código;
  • Sugerir melhorias na organização e na fluência do texto.

Além disso, algumas imagens e ilustrações presentes no livro foram geradas pelo Gemini, sempre com o objetivo de facilitar a compreensão de conceitos abstratos da área.

💡 Transparência: O uso de IA não substitui a revisão final do autor, nem a validação pelos colaboradores humanos. O material é constantemente testado e ajustado para garantir correção técnica e adequação didática. Acreditamos que a parceria entre autor e ferramentas de IA enriquece o processo de criação, desde que usada de forma ética e criteriosa.

Como este livro é produzido

Todo o conteúdo é escrito em Quarto, um sistema de publicação científica e técnica que permite renderizar o mesmo código-fonte para múltiplos formatos:

Formato Descrição
HTML Versão web completa, com navegação interativa, disponível em fzampirolli.github.io/pdi-vc
PDF Versão para impressão ou leitura offline
Notebooks para alunos Versão .ipynb processada por um filtro personalizado que resolve citações, numera figuras, tabelas e equações, e gera referências formatadas no estilo ABNT — ideal para uso no Jupyter e Google Colab

O filtro de pós-processamento (gerar_notebooks_alunos.py) prepara os notebooks para distribuição aos alunos, garantindo que funcionem perfeitamente em ambientes interativos mesmo sem depender do Quarto instalado.

Sobre as linguagens de programação e idiomas

O núcleo original deste livro é escrito em Português com exemplos em Python. Esta combinação (pt + py) é a versão de referência, mantida e revisada diretamente pelos autor.

Para ampliar o alcance do material, versões em outras combinações de idioma e linguagem de programação são geradas com o auxílio de APIs de Inteligência Artificial (modelos de tradução e conversão de código). Essas versões passam por um pipeline automatizado que:

  1. Traduz o texto explicativo para o idioma de destino
  2. Converte os exemplos de código Python para a linguagem alvo (ex: C++, R, Julia)
  3. Aplica o mesmo pós-processamento de citações, figuras, tabelas e equações

Exemplos de uso

O projeto utiliza um Makefile com atalhos para desenvolvimento. Veja alguns comandos comuns:

# Versão de referência: Português + Python, HTML + PDF
make build-all

# Apenas HTML (atualiza automaticamente ao salvar arquivos)
make html

# Português + C++, apenas PDF
make build-pdf LANGS=cpp LOCALES=pt

# Inglês + Python, apenas HTML
make build LANGS=py LOCALES=en

# Inglês + C++, HTML + PDF
make build-all LANGS=cpp LOCALES=en

# Todas as combinações disponíveis (py, cpp, java, c × pt, en, fr, es, it)
make full

# Gerar notebooks para alunos (distribuição Jupyter/Colab)
make alunos

# Publicar no GitHub Pages
make publish

💡 Dica: Você pode combinar qualquer idioma com qualquer linguagem. Os idiomas disponíveis são: pt (Português), en (Inglês), fr (Francês), es (Espanhol), it (Italiano). As linguagens disponíveis são: py (Python), cpp (C++), java (Java), c (C).

⚠️ Nota importante: As versões geradas por IA são um ponto de partida e podem conter imprecisões. Elas são disponibilizadas como material complementar e devem ser revisadas antes do uso em sala de aula. Contribuições da comunidade para refinar essas versões são muito bem-vindas!

Antes de começar: Notebooks em Python

Você já ouviu falar em Literate Programming (Programação Literária)? O conceito foi criado por Donald Knuth em 1984 (Knuth, 1984) — o mesmo que desenvolveu o sistema TeX para tipografia digital. Knuth iniciou esse projeto no final da década de 70 após ficar insatisfeito com a qualidade gráfica das novas edições de sua série clássica, The Art of Computer Programming (Knuth, 1997), buscando uma forma de unir a precisão matemática à beleza estética dos livros antigos. O conceito de Programação Literária não propõe apenas “combinar código e documentação”, mas sim inverter a lógica: o programa deve ser escrito para ser lido por humanos (como um ensaio), e as partes de código são extraídas dele para a máquina. É um detalhe sutil, mas que reforça o termo “literária”.

O conteúdo deste livro é formado por Notebook: um documento que intercala textos explicativos — as células de texto no formato Markdown — com códigos de programas em Python — as células de código. Mas aceita também outras linguagens de programação, mas o processo de execução é mais complicado.

É fácil distinguir as células de código: elas são precedidas por [ ]. Para executar uma célula de código, basta selecioná-la e pressionar Shift + Enter. Antes de começar, algumas observações importantes sobre o ambiente:

  • No computador: no Colab ou Jupyter, você também pode executar uma célula clicando no botão de ▶️ (play) que aparece ao passar o mouse sobre os colchetes [ ].
  • No celular: no Colab, o atalho Shift + Enter pode não funcionar. Nesse caso, toque no botão de ▶️ (play).
  • Execução local: você pode rodar este notebook no seu computador instalando o Jupyter Notebook.
  • Google Colab: para executar no seu navegador, vá em Arquivo → Salvar uma cópia no Drive e abra o arquivo Cópia de aula01.ipynb, que ficará na pasta Colab Notebooks do seu Drive.

O resultado será exibido logo abaixo da célula executada.
